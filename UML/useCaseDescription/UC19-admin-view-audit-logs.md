ตารางที่ ก.1.19  คำอธิบายยูสเคสการตรวจสอบบันทึกเหตุการณ์ (log) ที่เกิดขึ้นในระบบ

| หัวข้อ | รายละเอียด |
|---|---|
| รหัสยูสเคส | 19 |
| ชื่อยูสเคส | ตรวจสอบบันทึกเหตุการณ์ (log) ที่เกิดขึ้นในระบบ |
| ผู้เกี่ยวข้อง | ผู้ดูแลระบบ |
| ยูสเคสอ้างอิง | ระบบสารสนเทศสำหรับการจัดการข้อมูลทางการเกษตรผลผลิตยางพาราตามมาตรฐานจีเอพี |
| กระบวนการ | 1. ผู้ดูแลระบบเลือกเมนูตรวจสอบบันทึกเหตุการณ์ในระบบ<br>2. ระบบแสดงรายการเหตุการณ์ย้อนหลังทั้งหมดตามสิทธิ์การเข้าถึง<br>3. ผู้ดูแลระบบค้นหา กรอง และเรียงลำดับข้อมูลเหตุการณ์<br>4. ผู้ดูแลระบบเลือกเหตุการณ์เพื่อดูรายละเอียดก่อนและหลังการเปลี่ยนแปลง<br>5. ผู้ดูแลระบบตรวจสอบความผิดปกติหรือรายการที่ต้องติดตาม<br>6. ผู้ดูแลระบบใช้งานการดูจำนวนและลบข้อมูลเก่าตามนโยบายการเก็บรักษา (ถ้าจำเป็น) |
| กระบวนการ (ทางเลือก) | 1. หากไม่พบข้อมูลตามเงื่อนไขที่ค้นหา ระบบแสดงผลลัพธ์ว่าง<br>2. หากผู้ใช้ไม่มีสิทธิ์ผู้ดูแลระบบ ระบบไม่อนุญาตให้เข้าถึงบันทึกเหตุการณ์ |
| เงื่อนไขก่อน | 1. ผู้ดูแลระบบเข้าสู่ระบบเรียบร้อยแล้ว<br>2. ระบบมีข้อมูลบันทึกเหตุการณ์ให้ตรวจสอบ |
| เงื่อนไขหลัง | ผู้ดูแลระบบรับทราบสถานะเหตุการณ์ในระบบและสามารถดำเนินการกำกับดูแลต่อได้ |
